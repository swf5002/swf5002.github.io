Introduction

The first thing to do when presented with a problem is to construct a model that translates the problem into a mathematical context.

Algorithm
is a finite set of precise instructions for performing a computation or for solving a problem.

Properties shared by algorithms:

Input: An algorithm has input values from a specified set.
Output: From each set of input values an algorithm produces output values from a specified set.  
The output values are the solution to the problem.
Definite
or Precise:		 The steps of an algorithm must be defined precisely.
Correct: An algorithm should produce the correct output values for each set of input values.
Finite: An algorithm should produce the desired output after a finite (but perhaps large) number of steps for any input in the set.
Effective: It must be possible to perform each step of an algorithm exactly and in a finite amount of time.
Generality: The procedure should be applicable for all problems of the desired form, not just for a particular set of input values.

Searching Algorithms

The objective of a search algorithm is to determine if a certain element, say x, is among a list of elements.

Linear or Sequential Search
Compare x and a1.
When x = a1, then the solution is the location or subcript of a1, which is 1.
When x ≠ a1, then compare x with a2.
When x = a2, then the solution is the location or subcript of a2, which is 2.
When x ≠ a2, then compare x with a3.
. . . and so on until x is equal to one ai or x is not equal to any element on the list.

Binary Search
Start with a list in some order (either increasing or decreasing order; the example below assumes increasing).
Compare x with a middle term of the list, say an/2.
When x = an/2, then the solution is the location or subcript of an/2, which is n/2.
When x ≠ an/2, then determine if x < an/2 or if x > an/2.
If x < an/2, then compare x with an/4.
If x > an/2, then compare x with a3n/4.
. . . and so on until x is equal to one ai or x is not equal to any element on the list.

        then   // ( x ≠ ai )
            {
                location = 0 ;
            }

        return ;

The Halting Problem

This raises the question of whether, given a description of an arbitrary program and a set of inputs, it is possible to determine whether the program halts or not.   In 1936, Alan Turing proved using his definition of a Turing machine that it is impossible to tell. THE GROWTH OF FUNCTIONS


Introduction

This topic considers several aspects surrounding algorithms such as the number of operations used by algorithms,
the number of comparisons, the time required to solve a problem depending on hardware and software used.

The Big-O notation is used extensively to estimate the number of operations an algorithm uses as its input grows,
and thus determine whether or not it is practical to use a particular algorithm.


The Big-O Notation
Big-O Notation
Let f and g be functions from the set of integers or the set of real numbers to the set of real numbers.  
We say that f ( x ) is O ( g ( x ) ), read as "f ( x ) is big-oh of g ( x )",
if there are constants C and k such that | f ( x ) | ≤ C | g ( x ) | whenever x > k.
EXAMPLE06: Show that f ( x ) = x2 + 2x + 1 is O ( x2 ).
Observe than when x > 1, x < x2 and 1 < x2.
It follows that 0 ≤ x2 + 2x + 1 ≤ x2 + 2x2 + x2 = 4x2 whenever x > 1.
Consequently, C = 4 and k = 1 as witnesses to show that f ( x ) is O ( x2 ).
That is, f ( x ) = x2 + 2x + 1 < 4x2 whenever x > 1.
Observe than when x > 2, 2x ≤ x2 and 1 ≤ x2.
It follows that 0 ≤ x2 + 2x + 1 ≤ x2 + x2 + x2 = 3x2 whenever x > 2.
Consequently, C = 3 and k = 2 as witnesses to show that f ( x ) is O ( x2 ).
That is, f ( x ) = x2 + 2x + 1 < 3x2 whenever x > 2.

NOTE: If f ( x ) is O ( g ( x ) ) and g ( x ) is O ( f ( x ) ),
then we say that the two functions f ( x ) and g ( x ) are of the same order.

NOTE: If | f ( x ) | ≤ C | g ( x ) | for all x > k, and if | h ( x ) | > | g ( x ) | for all x > k,
then | f ( x ) | ≤ C | h ( x ) | for all x > k.

EXAMPLE06: Show that 7x2 is O ( x3 ).
When x > 7, multiplying both sides of the inequality by x2, we get x3 > 7x2.
Hence, 7x2 is O ( x3 ) when C = 1 and k = 7.
When x > 1, multiplying both sides of the inequality by 7x2, we get 7x3 > 7x2.
Hence, 7x2 is O ( x3 ) when C = 7 and k = 1.

THEOREM02: Let f ( x ) = anxn + an-1xn-1 + . . . + a1x1 + a0x0, where a0, a1, . . ., an are real numbers.  
Then f ( x ) is O ( xn ).
Proof: Recall that | x + y | ≤ | x | + | y |.   If x > 1, then
| f ( x ) | = | anxn + an-1xn-1 + . . . + a1x1 + a0x0 |
≤ | an | xn + | an-1 | xn-1 + . . . + | a1 | x1 + | a0 | x0
= xn ( | an | + | an-1 | / x + . . . + | a1 | / xn-1 + | a0 | / xn )
≤ xn ( | an | + | an-1 + . . . + | a1 | + | a0 )
This shows that | f ( x ) | ≤ C xn, where C = | an | + | an-1 | + . . . + | a1 | + | a0 | whenever x > 1.
Hence, f ( x ) is O ( xn ) when C = | an | + | an-1 | + . . . + | a1 | + | a0 | and k = 1.

EXAMPLE07: How can big-O notation be used to estimate the sum of the first n positive integers?
Consider 1 + 2 + 3 + . . . + n.
Since each of the integers in the sum of the first n positive integers does not exceed n,
it follows that 1 + 2 + 3 + . . . + n ≤ n + n + n + . . . + n = n * n = n2.
Therefore, 1 + 2 + 3 + . . . + n is O ( n2 ) where C = 1 and k = 1.

THEOREM03: Suppose that f1 ( x ) is O ( g1 ( x ) ) and f2 ( x ) is O ( g2 ( x ) ).  
Then ( f1 + f2 ) ( x ) is O ( max ( | g1 ( x ) |, | g2 ( x ) | ) ).
Proof: Suppose that f1 ( x ) is O ( g1 ( x ) ) and f2 ( x ) is O ( g2 ( x ) ).
From the definition of big-O notation, there are constants C1, C2, k1, k2 such that
| f1 ( x ) | ≤ C1 | g1 ( x ) |, where x > k1, and | f2 ( x ) | ≤ C2 | g2 ( x ) |, where x > k2.
To estimate the sum of f1 ( x ) and f2 ( x ), note that
| ( f1 + f2 ) ( x ) | = | f1 ( x ) + f2 ( x ) |
≤ | f1 ( x ) | + | f2 ( x ) | using the triangle inequality | a + b | ≤ | a | + | b |
≤ C1 | g1 ( x ) | + C2 | g2 ( x ) | where x > k1 and x > k2
≤ C1 | g ( x ) | + C2 | g ( x ) |
= ( C1 + C2 ) | g ( x ) | where g ( x ) = max ( g1 ( x ), g2 ( x ) )
= C | g ( x ) | where C = C1 + C2
Therefore, | ( f1 + f2 ) ( x ) | ≤ C | g ( x ) |, whenever x > k, where k = max ( k1, k2 )

THEOREM04: Suppose that f1 ( x ) is O ( g1 ( x ) ) and f2 ( x ) is O ( g2 ( x ) ).  
Then ( f1 f2 ) ( x ) is O ( g1 ( x ) g2 ( x ) ).
Proof: Suppose that f1 ( x ) is O ( g1 ( x ) ) and f2 ( x ) is O ( g2 ( x ) ).
From the definition of big-O notation, there are constants C1, C2, k1, k2 such that
| f1 ( x ) | ≤ C1 | g1 ( x ) |, where x > k1, and | f2 ( x ) | ≤ C2 | g2 ( x ) |, where x > k2.
To estimate the product of f1 ( x ) and f2 ( x ), note that
| ( f1 f2 ) ( x ) | = | f1 ( x ) f2 ( x ) |
= | f1 ( x ) | | f2 ( x ) |
≤ C1 | g1 ( x ) | C2 | g2 ( x ) | where x > k1 and x > k2
≤ C1 C2 | g1 ( x ) g2 ( x ) |
≤ C | g1 ( x ) g2 ( x ) | where C = C1 C2
Therefore, | ( f1 f2 ) ( x ) | ≤ C | g1 ( x ) g2 ( x ) |, whenever x > k, where k = max ( k1, k2 )

The Big-Omega and Big-Theta Notation
Big–Omega Notation
Let f and g be functions from the set of integers or the set of real numbers to the set of real numbers.  
We say that f ( x ) is Ω ( g ( x ) ), read as "f ( x ) is big–Omega of g ( x )", if there is a positive constant C and k such that | f ( x ) | ≥ C | g ( x ) | whenever x > k.
Big–Theta Notation
Let f and g be functions from the set of integers or the set of real numbers to the set of real numbers.  
We say that f ( x ) is Θ ( g ( x ) ), read as "f ( x ) is big–Theta of g ( x )", if f ( x ) is O ( g ( x ) ) and f ( x ) is Ω ( g ( x ) ).  
When f ( x ) is Θ ( g ( x ) ), we say that f ( x ) is of order g ( x ).
SUMMARY: Big–O: f(x) is O(g(x)), if there are constants C and k such that |f(x)| ≤ C|g(x)| whenever x>k.
Big–Ω: f(x) is W(g(x)), if there is a positive constant C and k such that |f(x)| ≥ C|g(x)| whenever x>k.
Big–Θ: f(x) is Θ(g(x)), if there is a positive constant C and k such that |f(x)| = C|g(x)| whenever x>k.

COMPLEXITY OF ALGORITHMS


Introduction

Efficiency of algorithms can be measured in terms of the computer time to solve a problem using the algorithm when input values are of a specified size, or in terms of the computer memory required to implement the algorithm when input values are of a specified size.   These involve the computational complexity of the algorithm.   An analysis of computer time involves the time complexity of the algorithm while an analysis of computer memory involves the space complexity of the algorithm.

Space complexity is tied to data structures which are not covered in this course, so space complexity will not be covered.


Time Complexity

Time complexity of an algorithm can be expressed in terms of the number of operations used by the algorithm when the input has a particular size.   The complexity is described in terms of the number of operations required instead of actual computer time because of the differences in time needed for different computers to perform basic operations.

Division
Division
If a and b are integers with a ≠ 0, we say that a divides b if there is an integer c such that b = ac.
When a divides b, we say that a is a factor of b and that b is a multiple of a.
The notation a | b denotes that a divides b.
EXAMPLE14: Determine if 3 | 15 and if 7 | 19.
Since 15 / 3 = 5 is an integer, therefore 3 | 15.

Since 19 / 7 is not an integer, therefore it is NOT the case that 7 | 19.

THEOREM06: Let a, b, and b be integers.   Then
  1. if a | b and a | c, then a | ( b + c )
  2. if a | b, then a | bc for all integers c
  3. if a | b and b | c, then a | c
Proof: i Suppose a | b and a | c.
By definition, it follows that there are integers s and t, with b = as and c = at.
Hence, b + c = as + at = a ( s + t ).
Therefore, a | b + c.
ii Suppose a | b.
By definition, it follows that there is an integers s, with b = as.
Multiplying both sides of the equation, we get bc = a ( sc ).
Therefore, a | bc.
iii Suppose a | b and b | c.
By definition, it follows that there are integers s and t with b = as and c = bt.
Then, c = bt = ast.
Hence, c = a ( st ).
Therefore, a | c.

COROLLARY06-01: If a, b, and c are integers such that a | b and a | c,
then a | mb + nc whenever m and n are integers.
Proof: Since a | b, by THEOREM06.ii, it follows that a | bm or a | mb, for some integer m.
And since a | c, by THEOREM06.ii, it follows that a | cn or a | nc, for some integer n.
By THEOREM06.i, it follows that a | mb + nc.

Modular Arithmetic
Congruent Modulo
If a and b are integers and m is a positive integer, then a is congruent to b modulo m if m divides a – b.  
We use the notation a ≡ b ( mod m ) to indicate that a is congruent to b modulo m.
THEOREM08: Let a and b be integers, and let m be a positive integer.  
Then a ≡ b ( mod m ) iff a mod m = b mod m.
Proof: If a ≡ b ( mod m ), then by definition, m | ( a – b ).
This means that there is an integer s such that a – b = sm.
Hence, a = b + sm, which also means that a mod m = b.
This also means that b = a + ( –s ) m, which also means that b mod m = a.
It also follows that m | ( b – a ).
Hence, b ≡ a ( mod m ).
If a mod m = b mod m = r, then for some integer q, a = qm + r = b.
This means that a – r = qm.
Similarly, this also means that b – r = qm.

EXAMPLE16: Determine where 17 is congruent to 5 modulo 6 and whether 24 and 14 are congruent modulo 6.
Since 6 | 17 – 5 = 12, then 17 ≡ 5 ( mod 6 ).

Since 24 – 14 = 10 is not divisible by 6, then 24 is not congruent to 14 ( mod 6).

THEOREM09: Let m be a positive integer.  
The integers a and b are congruent modulo m iff there is an integer k such that a = b + km.
Proof: If a ≡ b ( mod m ), then by definition m | ( a – b ).
This means that there is an integer k such that a – b = km.
Therefore, a = b + km.
Conversely, if there is an integer k such that a = b + km, then km = a – b.
Hence, m | ( a – b ).
Therefore, a ≡ b ( mod m ).

THEOREM10: Let m be a positive integer.  
If a ≡ b ( mod m ) and c ≡ d ( mod m ), then a + c ≡ ( b + d ) ( mod m ) and ac ≡ bd ( mod m )
Proof: If a ≡ b ( mod m ), then there is an integer j such that a = b + jm.
This is also equivalent to – b = – a + jm.
This is equivalent to – ( – b = – a + jm ).
This is equivalent to b = a – jm.
Let s = – j.
Then b = a + sm.
Similarly, if c ≡ d ( mod m ), then there is an integer k such that c = d + km.
This is also equivalent to – d = – c + km.
This is equivalent to – ( – d = – c + km ).
This is equivalent to d = c – km.
Let t = – k.
Then d = c + tm.
b + d = ( a + sm ) + ( c + tm ) = ( a + c ) + m ( s + t )
Hence, a + c ≡ ( b + d ) ( mod m ).
Likewise, bd = ( a + sm ) ( c + tm ) = ac + m ( at + sc + stm )
Hence, ac ≡ ( bd ) ( mod m ).

Base Conversion Algorithm
  • Divide n by b to obtain a quotient and a remainder, i.e. n = bq0 + a0 where 0 ≤ a0 < b.
  • The remainder a0 is the rightmost digit in the base b expansion of n.
  • Divide q0 by b to obtain q0 = bq1 + a1 where 0 ≤ a1 < b.
  • a1 is the second digit from the right in the base b expansion of n.
  • Continue the process, successively dividing the quotients by b, obtaining additional base b digits as the remainders.
  • This process terminates when we obtain a quotient equal to 0.

In cryptography, it is important to be able to find bn mod m efficiently, where b, n, and m are large integers.  
It is therefore impractical to first compute bn and then find its remainder when divided by m because bn will be a huge number.

n = ( ak-1 . . . a1a0 )2

bn = bak-12k-1+...+a121+a020 = bak-12k-1 * ba121 * ba020

Then find

b20 mod m, b21 mod m, b22 mod m, . . ., b2k-1 mod m

and multiply together the terms b2k mod m where ak = 1,
finding the remainder of the product when divided by m after each multiplication.

function gcd ( int a, int b )
    {
        int r, x, y ;

        x = a ;
        y = b ;

        while ( y ≠ 0 )

    }

THEOREM21: Fermat's Little Theorem
If p is prime and a is an integer not divisible by p, then ap-1 ≡ 1 ( mod p ).  
Furthermore, for every integer a we have ap ≡ a ( mod p ).

The RSA Cryptosystem
RSA Cryptosystem
is based on modular exponentiation modulo the product of two large prime numbers.  
Each individual has an encryption key consisting of a modulus n = pq,
where p and q are large prime numbers and an exponent e that is relatively prime to ( p – 1 ) (q – 1 ).
To produce a usable key, two large prime numbers, with say 200 or 400 digits,
must be found which can be quickly done on a computer using a probabilistic primality test.

RSA Encryption
RSA Encryption
is the method where messages are translated into sequences of integers by translating each letter into an integer.  
The integers are grouped together to form larger integers, each representing a block of letters.  
The encryption proceeds by transforming the integer M, representing the plaintext (original message) to an integer C, representing the ciphertext (encrypted message), using the function C = Me mod n.
