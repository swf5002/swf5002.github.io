Tree
is a connected undirected graph with no simple circuits.
THEOREM01: An undirected graph is a tree iff there is a unique simple path between any two of its vertices.
Root
of a tree T is a designated vertex v0 and every edge is directed away from it.

Properties of Trees
THEOREM02: A tree with n vertices has n–1 edges.

THEOREM03: A full k-tree with i internal vertices contains n = ki + 1 vertices.

THEOREM04: A full k-tree with
  1. n vertices has i = ( n – 1 ) / k internal vertices and l = [ ( k – 1 )n + 1 ] / k leaves.
  2. i internal vertices has n = ki + 1 vertices and l = ( k – 1)i + 1 leaves.
  3. l leaves has n = ( kl – 1 ) / ( k – 1 ) vertices and i = ( l – 1 ) / ( k – 1 ) internal vertices.

THEOREM05: There are at most kh leaves in a k-tree of height h.

COROLLARY05-01: If a k-tree of height h had l leaves, then h ≥ ⌈ logkl ⌉.   If the k-tree is full and balanced, then h = ⌈ logkl ⌉.

ALGORITHM01: Balanced Binary Tree Sort Algorithm
  1. Make the first data the root of the tree.
  2. If the next data is less than the value of the current vertex, then take the left path, otherwise take the right path, and add a new vertex to the leaf at the end of the path for the new data.
  3. Recalculate the heights and balances of the vertices along the path from the leaf towards the root until an imbalance is encountered or the root is reached.
  4. Rebalance if an imbalance is encountered and recalculate the heights and balances of the affected vertices.
  5. Go to step 2 and repeat the process until all data have been sorted and stored in tree.

ALGORITHM02: Huffman Coding
function Huffman ( C: symbols ai with frequencies wi, i = 1, 2, . . ., n )
    {
        F = forest of n rooted trees, each consisting of the single vertex ai and assigned weight wi

        while ( F is not a tree )
            {
                Replace rooted trees T and T' of least weights from F with w ( T ) ≥ w ( T' )
                with a tree having a new root that has T as a its left subtree and T' as its right subtree.
                Label the new edge to T with 0 and the new edge to T' with 1.
                Assign w ( T ) + x ( T' ) as the weight of the new tree.
            }
    }

Universal Address Systems
Universal Address System
is an algorithm describing a way to totally order the vertices of a rooted tree.
  1. LEVEL 0:   Label the root with the integer 0.
  2. LEVEL 1:   Label the k children of the root from left to right with 1, 2, . . ., k.
  3. LEVEL n:   Label the kv children of each vertex v labelled A at level n from left to right with A.1, A.2, . . ., A.kv.

Infix, Prefix, Postfix Notation
Expression
consists of two components namely operands and operators.
Operator
indicate the operation to be carried out on operands.

There are two kinds of operators namely unary and binary.

Unary Operator
requires one operand to carry out the intended operation
Binary Operator
requires two operands to carry out the intended operation.

Most operators are binary.

Notation
There are three ways of representing expressions:
  • In prefix notation, the operator is written before the operand/s it operates on.
  • In infix notation, the operator is written in between the operands it operates on as in the case of binary operators.
  • In postfix notation, the operator is written after the operand/s it operates on.

Regardless of the notation, an expression is represented in a binary tree in the same manner all the time.
Moreover, a tree representing an expression contains no parentheses.

Breadth-First Search
ALGORITHM07: Breadth-First Search
function BFS ( G: connected graph with vertices v1, v2, . . ., vn )
    {
        T = tree consisting only of vertex v1 ;
        L = empty list ;
        put v1 in the list L of unprocessed vertices ;

        while ( L is not empty )
            {
                remove the first vertex v from L ;

                for ( each neighbor w of v )
                    {
                        if ( w is not in L and not in T )
                            {
                                add w to the end of the list L ;
                                add w and edge { v, w } to T ;
                            }
                    }
            }
    }
