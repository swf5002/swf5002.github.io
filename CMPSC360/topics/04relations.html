RELATIONS AND THEIR PROPERTIES


Introduction
Binary Relation
R from set A to set B is a subset of A x B consisting of a set of ordered pairs R = { ( a, b ) | ( a ∈ A ) ∧ ( b ∈ B ) }.   The notation aRb or R(a,b) denotes that ( a, b ) ∈ R.
Domain
of relation R is the set A (or a subset of A) where R is a relation from A to B.
Range
of relation R is the set B where R (or a subset of B) is a relation from A to B.

Relations on a Set
Relation
on a set S from S to S is a subset of S x S. Unless otherwise stated, the elements of a binary relation will be composed of ordered pairs, and each element of the ordered pairs will be elements of the set S.


Properties of Relations
Reflexive
A relation R on a set S, i.e., R ⊆ S x S, is reflexive iff

∀x ( ( x, x ) ∈ R )

The quantifiers in these definitions range over elements of the set S. An example of reflexive relation on numbers is =.

Irreflexive
A relation R on a set S, R ⊆ S x S, is irreflexive iff

∀x ( ( x, x ) ∉ R )

An example of an irreflexive relation on numbers is < or >.
Symmetric
A relation R on a set S, i.e. R ⊆ S x S, is symmetric iff

∀x ∀y ( ( x, y ) ∈ R → ( y, x ) ∈ R )

An example of a symmetric relation on numbers is =.

Antisymmetric
A relation R on a set S, i.e. R ⊆ S x S, is antisymmetric iff

∀x ∀y ( ( ( x, y ) ∈ R ∧ ( y, x ) ∈ R ) → x = y )

An example of an antisymmetric relation on numbers is ≤. An example of an antisymmetric relation on sets is ⊆, as A ⊆ B and B ⊆ A implies A = B.
Asymmetric
A relation R on a set S, R ⊆ S x S, is asymmetric iff

∀x ∀y ( ( x, y ) ∈ R → ( y, x ) ∉ R )

An example of an asymmetric relation on numbers is >.
Transitive
A relation R on a set S, i.e. R ⊆ S x S is transitive iff

∀x ∀y ∀z ( ( ( x, y ) ∈ R ∧ ( y, z ) ∈ R → ) ( x, z ) ∈ R )

Examples of transitive relations on numbers include = and >.


Combining Relations
Composition of Relations
The composite of relations R and T, denoted by T ∘ R is defined as:
  1. If a ∈ A, b ∈ B, and c ∈ C (A, B, and C may the the same set); and
  2. If R is a relation from A to B: R = { (a, b) | a ∈ A ∧ b ∈ B }; and
  3. If T is a relation from B to C: R = { (b, c) | b ∈ B ∧ c ∈ C ; then
  4. T ∘ R = { (a, c) | ∃b ( (a, b) ∈ R ∧ (b, c) ∈ T ) }
EXAMPLE01: Let R be a relation from { 0, 1, 2 } to { 4, 6, 8 } where R = { (0, 4), (0, 6), (0, 9), (1, 6), (2, 4), (2, 8) }.
Let T be a relation from { 6, 9 } to { 3, 5, 7 ) where T = { (6, 3), (9, 3), (9, 7)}.
T ∘ R = { (0, 3), (0, 7), (1, 3) }.

EXAMPLE02: Let R be the relation on the set of all people such that (a, b) ∈ R if a is a parent of b.
Then (a, c) ∈ R ∘ R, if (b, c) ∈ R.   This means that a is a grandparent of c.
Rn
The power Rn, for n = 1, 2, 3, . . . is defined recursively by R1 = R, and Rn+1 = Rn ∘ R where R is a relation on the set A x B.

Note that the definition of a transitive relation and part 4 of the definition of the composite of two relations are very similar, but still different. The composite of R with itself, or Rn for any finite n, must be transitive (as THEOREM01 below demonstrates), if and only if Rn ⊆ R1 = R. And, Rn ⊆ R if and only if (a, b) ∈ Rn → (a, b) ∈ R.

N-ary RELATIONS AND THEIR APPLICATIONS


Introduction

Relationships among elements of more than two sets often arise and are referred to as N-ary relations.


n-ary Relations
N–ary Relation
R between the elements of n sets, S1, S2, . . ., Sn, is a subset of the Cartesian product of those n sets, i.e.

R ⊆ ( S1 x S2 x . . . x Sn ) = { (x1, x2, . . . , xn) | x1 ∈ S1, x2 ∈ S2, . . ., xn ∈ Sn }.

S1, S2, . . ., Sn is called the domain while n is called its degree. An element of R is an n-tuple.
EXAMPLE04: Let R be a relation on N=id Number x S=Student name x M=Major x A=gpA x Y=graduation Year, i.e., R ⊆ S x N x M x A x Y.

Databases and Relations

A relational database consists of n-tuple records made up of fields.

Database records (n-tuples, also called rows) are stored in a table.   Each column of the table corresponds to an attribute, for which there are sets of admissible values. That is, the value of each attribute is drawn from a domain of admissible values.

If a subset of the attributes uniquely identifies a row, that subset of attributes is called a primary key.

EXAMPLE05:
Table: StudentRecord
N S M A Y
123456789 Cappuccio MATH 3.33 1970
234567890 Arons MGMT 3.24 1969
345678901 Peredo ACCTG 3.69 1971
456789012 Donato MKTG 3.48 1974
Since values of S, M, A, and Y may occur more than once, the logical choice for primary key is unique N.

Records are often added or deleted from databases.   Hence, the primary key is time-dependent.   The current collection of n-tuples in a relation, stored in a table, is called the extension of the relation.   A more permanent part of a database, including the name and attributes of the database tables, is called its intension, or schema.   A primary key (if there is one) must be able to uniquely identify each row in the extension of a relation. A primary key of a table is declared in the table's schema definition.

The database might, then contain another table, the StudentData table, with the same primary key (student ID number), and attributes such as: address, telephone number, and so on.


Operations on n-ary Relations

Database queries are used to retrieve information from a database; that information may be stored in a single table, but more often than not the information sought is stored in multiple tables. Retrieving information in multiple tables involves creating new n-ary relations (specified in the query, based on conditions also specified in the query).  The most basic operation on an n-ary relation in a database is determining all the n-tuples that satisfy a given condition.

Selection Operation
Sc, selects from the n-ary relation R (in a single table) all n-tuples that satisfy the condition C. A selection, for instance, might involve retrieving all n-tuples in the StudentRecord table that satisfy the condition: Major="MKTG".
Projection Operation
Pi1i2...im, where i1 < i2 < . . . < im, maps the n-tuple ( a1, a2, . . ., an ) to the m-tuple ( ai1, ai2, . . ., aim ), where m ≤ n.  

This retrieves only n – m of the components (attributes) of an n-tuple (stored in a single table). A projection on the StudentRecord table might only specify the student ID and gpa attributes of every row in the table.

Join Operation
Jp( R, T ), where R is a relation with degree m, T is a relation with degree n, p ≤ m and p ≤ n, is a relation of degree m + n – p that consists of all (m + n – p)-tuples ( a1, a2, . . ., am-p, c1, c2, . . ., cp, b1, b2, . . ., bn-p ) where the m-tuple ( a1, a2, . . ., am-p, c1, c2, . . ., cp ) belongs to R and the n-tuple ( c1, c2, . . ., cp, b1, b2, . . ., bn-p ) belongs to T.   This means that Jp produces a new relation from the two relations by combining all the m-tuples of the first relation with all n-tuples of the second relation, where the last p components of the m-tuples agree with the first p components of the n-tuples.

A join is, then, a composite relation as defined above. A example might be a retrieval from the StudentRecord and StudentData tables that gets only the student names and telephone numbers of all marketing majors with gpa's greater than 3.0.


SQL

Representing Relations Using Digraphs
Graph
G = ( V, E ) ( or G = < N, E > ) is a set of finite and non-empty vertices V = { v0, v1, . . ., vk} ( or nodes ) and a set of finite and non-empty edges E = { e1, e2, . . ., ej} (or arcs), where an edge denoted as an ordered pair ( a, b ) connects two vertices a and b.  
Directed Graph or Digraph
is a graph G = ( V, E ) with edge ( a, b ) where a is the initial vertex while b is the terminal vertex.
EXAMPLE14: Consider the relation R on A = { a, b, c, d } where R = { ( a, b ), ( a, d ), ( b, b ), ( b, d ), ( c, a ), ( c, b ), ( d, b ) }.
The digraph representing R will have vertices labeled with the elements of A and edges representing the elements of R.

CLOSURES OF RELATIONS


Introduction
Closure
Let R be a relation on A.   R may or may not have some property P, such as reflexivity, symmetry, or transitivity.   If there is a relation S with property P containing R such that S is a subset of every relation with property P containing R, then S is called the closure of R with respect to P.

Closures
Reflexive Closure
If a reflexive relation S contains a relation R, then S is the reflexive closure of R.
Symmetric Closure
If a symmetric relation S contains a relation R, then S is the symmetric closure of R.
EXAMPLE15: Given R = { ( a, b ) | a > b } on the set of positive integers.
What is the reflexive closure of R?
The reflexive closure of R is
S = R ∪ { ( a, a ) | a ∈ Z+ } = { ( a, b ) | a ≥ b }
What is the symmetric closure of R?
The symmetric closure of R is
S = R ∪ { ( b, a ) | a > b } = { ( a, b ) | a ≠ b }

Observe that is producing the closure set, S, we need only add the ordered pairs that are missing in R that are needed to satisfy the reflexive or symmetric closure condition.

The construction of a transitive closure is not as simply defined as the construction of a reflexive closure and a symmetric closure. The transitive closure of a relation is defined more precisely below. But, we can see intutively that the transitive closure of a relation { (a, b), (b, c) } is { (a, b), (b, c), (a, c) }. And, the transitive closure of a relation { (a, b), (b, c), (c, d) } is { (a, b), (b, c), (c, d), (a, c), (a, d), (b, d) }.


Paths in Directed Graphs
Path
from a to b in the digraph G is a sequence of edges ( x0, x1 ), ( x1, x2 ), ( x2, x3 ), . . . , ( xn-1, xn ) in G, where n is a non-negative integer, and x0 = a and xn = b.   This path may also be denoted as x0, x1, x2, x3, . . . , xn and had length n. Note that n counts only the number of edges (ordered pairs), not the number of nodes (which is n+1).
Circuit or Cycle
is a path of length n ≥ 1 that begins and ends at the same vertex.

Note that a path in a digraph can pass through a vertex more than once and an edge in a digraph can occur more than once in a path.

There is a path from a to b in R if there is a sequence of elements a, x1, 2, x3, . . . , xn-1, b with ( a, x1 ) ∈ R, ( x1, x2 ) ∈ R, ( x2, x3 ) ∈ R, . . . , ( xn-1, b ) ∈ R.

THEOREM02: Let R be a relation on a set A.   There is a path of length n, where n is a positive integer, from a to b iff ( a, b ) ∈ Rn.
Proof: (by induction)

By definition, there is a path from a to b of length 1 iff ( a, b ) ∈ R.
Therefore, the theorem is T for n = 1.

Assume the theorem is T for n > 1.
There is a path of length n + 1 from a to b iff there is an element c ∈ A such that
there is a path of length 1 from a to c so ( a, c ) ∈ R, and a path of length n from c to b so ( c, b ) ∈ Rn.
And there is an element c with ( a, c ) ∈ R and ( c, b ) ∈ Rn iff ( a, b ) ∈ Rn+1.

THEOREM02 connects the definition of the power of a relation (Rn, which is a composition of relation R with itself) to the definition of a path of length n.

In the context of digraphs, the transitive closure of the relation defining the digraph includes all ordered pairs of nodes, (ni, nj), where there is a path path of any length from ni to nj. Hence, the connectivity map of a digraph is the transtive closure of the relation R defining the digraph.

Because Rn consists of the pairs ( a, b ) such that there is a path of length n from a to b, it follows that R* is the union of all the sets Rn.

Warshall's Algorithm
Interior Vertices
If a, x1, x2, x3, . . ., xm-1, b is a path, then its interior vertices are x1, x2, x3, . . ., xm-1, that is, all the vertices of the path except the first and last vertices.   The first vertex in the path is not an interior vertex unless it is visited again by the path, except as the last vertex.   Similary, the last vertex in the path is not an interior vertex unless it is visited again by the path, except as the first vertex.  
Warshall's Algorithm
is an efficient method for computing the transitive closure in a relation.   It is based on the construction of a sequence of 0-1 matrices W0, W1, W2, . . . , Wn, where W0 = MR and Wn = MR*.
LEMMA04-01: Let Wk = [ wij[k]] be the 0-1 matrix that has a 1 in its (i, j)th position iff there is a path from vi to vj with interior vertices from the set { v1, v2, v3, . . ., vk }.
Then wij[k] = wij[k-1] ∨ ( wik[k-1] ∧ wkj[k-1] ), whenever i, j, and k are positive integers not exceeding n.

EQUIVALENCE RELATIONS


Equivalence Relations
Equivalence Relation
A relation R is an equivalence relation if it is reflexive, symmetric, and transitive.
Equivalent
Two elements a and b that are related by an equivalence relation are equivalent with respect to a particular equivalence relation and is denoted by a ~ b.
EXAMPLE18: Let R be the relation on the set of real numbers such that aRb iff a – b is an integer.   Is R an equivalence relation?
Since for any real number a, a – a = 0 is an integer, then aRa for all real numbers a.
Hence, R is reflexive.
Suppose aRb, then a – b = x is an integer, and b – a = – x is also an integer.
Hence, bRa and it follows that R is symmetric.
Suppose aRb and bRc, then a – b and b – c are both integers.
And ( a – b ) + ( b – c ) is also an integer, since the sum of two integers is an integer.
But ( a – b ) + ( b – c ) = a – b + b – c = a – c is also an integer.
Hence, aRc and it follows that R is transitive.
Since R is reflexive, symmetric, and transitive, it follows that R is an equivalence relation.
Congruence Modulo
If n > 1 is a positive integer, then two integers a and b are congruent modulo n if n | (a – b), i.e. if n divides a – b (evenly).   Congruence modulo is denoted by a ≡ b (mod n).   We also say that a is congruent to b modulo n.

R = { ( a, b ) | a ≡ b ( mod n ) }
Alternatively, modulo congruence can be defined in terms of the arithmetic % operator on integers, where a % n = the remainder when dividing integer a by positive integer n. The remainders can range from 0 to n-1. Then two integers, a and b, are congruent modulo n, a ≡ b (mod n) iff a % n = b % n.
EXAMPLE19: Is R = { ( a, b ) | a ≡ b ( mod n ) } an equivalence relation?
Consider ( a, a ) where a – a = 0 = 0 * n.
Therefore, n | ( a – a ) and a ≡ a ( mod n ).
Hence, congruence modulo is reflexive.
Suppose a ≡ b ( mod n ), then n | ( a – b ) and a – b = k * n where k is an integer.
It follows that b – a = – k * n, so b ≡ a ( mod n ).
Hence, congruence modulo is symmetric.
Suppose a ≡ b ( mod n ) and b ≡ c ( mod n ), then n | ( a – b ) and n | ( b – c ).
It follows that a – b = k * n and b – c = m * n, where both k and m are integers.
Therefore, a – c = ( a – b ) + ( b – c ) = k * n + m * n = ( k + m ) n.
Thus, a – c = ( k + m ) n implies that n | ( a – c ) and a ≡ c ( mod n ).
Hence, congruence modulo is transitive.
Since congruence modulo is reflexive, symmetric, and transitive, it follows that congruence modulo is an equivalence relation.

Equivalence Classes
Equivalence Class
If R is an equivalence relation on a set S, and x ∈ S, then the equivalence class of x is the set of elements y such that xRy and where y ∈ S, i.e.

[ x ] = { y | xRy ∧ x ∈ S ∧ y ∈ S }
EXAMPLE20: What are the equivalence classes of 0 and 1 for congruence modulo 4?
The equivalence class of 0 contains all integers x such that x ≡ 0 ( mod 4 ), i.e. the integers divisible by 4.
Hence, the equivalence class of 0 is [ 0 ]4 = { . . ., –12, –8, –4, 0, 4, 8, 12, . . . }
The equivalence class of 1 contains all integers y such that y ≡ 1 ( mod 4 ), i.e. the integers with remainder 1 when divided by 4.
Hence, the equivalence class of 1 is [ 1 ]4 = { . . ., –11, –7, –3, 1, 5, 9, 13, . . . }

Generalizing EXAMPLE20, the equivalence classes of the relation congruence modulo n called congruence classes modulo n and denoted by [ a ]n is:

[ a ]n = { . . ., a – 2n, a – n, a, a + n, a + 2n, . . . }

Equivalence Classes and Partitions
THEOREM05: Let R be an equivalence relation on a set A with a ∈ A, b ∈ A, and a ≠ b.   The following statements are equivalent:
  1. aRb
  2. [ a ] = [ b ]
  3. [ a ] ∩ [ b ] ≠ ∅
Proof: Suppose aRb.
If c ∈ [ a ], then aRc.
It follows from symmetry that since aRb, then bRa.
It follows from transitivity that since bRa and aRc, then bRc.
Therefore, c ∈ [ b ] and [ a ] ⊆ [ b ].
It also follows from transitivity that since aRb and bRc, then aRc
Therefore, [ b ] ⊆ [ a ]
Thus, [ a ] = [ b ].
Hence, (i) implies (ii).
If [ a ] = [ b ], it follows that [ a ] ∩ [ b ] ≠ ∅ because [ a ] is non-empty (it contains at least a and b).
Hence, (ii) implies (iii).
If [ a ] ∩ [ b ] ≠ ∅, it follows that ∃ c, such that c ∈ [ a ] and c ∈ [ b ].
Therefore, aRc and bRc.
It follows from bRc that by symmetry, cRb.
It follows from aRc and cRb that by transitivity, aRb.
Hence, (iii) implies (i).
Since (i) implies (ii), and (ii) implies (iii), and (iii) implies (i), the three statements are equivalent.

OBSERVATION: Let R be an equivalence relation on a set A.   The union of the equivalence classes of R is all of A, because each a ∈ A is in its some equivalence class, namely [ a ]R, i.e.
∪ [ a ]R = A
a∈A

From THEOREM05, it follows that these equivalence classes are either equal or disjoint, i.e.

[ a ]R ∩ [ b ]R = ∅ iff [ a ]R ≠ [ b ]R

Therefore, the equivalence classes form a partition of A because they split A into disjoint subsets.  

Partition
of a set S is a collection of disjoint non-empty subsets of S that has S as their union.

In other words, the collection of subsets Ai, i ∈ I, where I is an index set, forms a partition of S iff

i. Ai ≠ ∅ for i ∈ I
ii. Ai ∩ Aj = ∅ when i ≠ j
iii.
∪ Ai = S
i∈I

THEOREM06: Let R be an equivalence relation on a set S.   Then the equivalence classes of R form a partition of S.  
Conversely, given a partition { Ai | i ∈ I } of the set S, there is an equivalence relation R that has the sets Ai, i ∈ I, as its equivalence classes.

PARTIAL ORDERING


Introduction
Partial Ordering
A relation R on a set S is a partial ordering or partial order is it is reflexive, antisymmetric, and transitive.
Partially Ordered Set
or poset is a pair ( S, R ), where S is a set and R is a partial order on S.   The members of S are called elements of the poset.
N–Cube
The poset ( Þ(S), ⊆ ) where S = { 1, 2, . . ., n }. Recall that the power set of a set, S, is the set of all subsets of S.
EXAMPLE22: If S is any set and Þ(S) is its power set, then the relation of ⊆ is a partial ordering of Þ(S).
Proof: Whenever A ⊆ Þ, then A ⊆ A.
Therefore, ⊆ is reflexive.
Whenever A ⊆ Þ and B ⊆ Þ,
if A ⊆ B and B ⊆ A, then A = B.
Therefore, ⊆ is antisymmetric.
Whenever A ⊆ Þ, B ⊆ Þ, and C ⊆ Þ,
if A ⊆ B and B ⊆ C, then A ⊆ C.
Therefore, ⊆ is transitive.
Hence, ( Þ(S), ⊆ ) is a poset.
Comparable
The elements a and b of a poset ( S, ≽ ) are comparable if either a ≽ b or b ≽ a.  
When a and b are elements of S such that neither a ≽ b nor b ≽ a, a and b are called incomparable.
Totally Ordered Set or Linearly Ordered Set
If ( S, ≽ ) is a poset and every two elements of S are comparable, >S is said to be totally ordered set or linearly ordered set and ≽ is called a total order or a linear order.
A totally ordered set is also called a chain.
Well-Ordered Set
( S, ≽ ) is a well-ordered set if it is a poset such that ≽ is a total ordering and every non-empty subset of S has a least element.

Lexicographic Order
Lexicographic Ordering
≺ on A1 x A2 is defined by specifying that one pair is less than a second pair if the first entry of the first pair is less than (in A1) the first entry of the second pair, or if the first entries are equal, but the second entry of this pair is less than (in A2) the second entry of the second pair.  
In other words, ( a1, a2 ) ≺ ( b1, b2 ), either if a1 ≺1 b1 or if both a1 = b1 and a2 ≺2 b2.
Lexicographic Ordering of Strings
Consider the strings a1a2 . . . am and b1b2 . . . bn on a partially ordered set S.   Suppose these strings are not equal.   Let t be the minimum of m and n.   The definition of lexicographic ordering is that the string a1a2 . . . am is less than b1b2 . . . bn iff ( a1, a2, . . ., at ) ≺ ( b1, b2, . . ., bt ), or ( a1, a2, . . ., at ) = ( b1, b2, . . ., bt ) and m < n, where ≺ represents the lexicographic ordering of St.
In other words, to determine the lexicographic ordering of two different strings, truncate the longer string to the length of the shorter string, i.e. t = min ( m, n ) terms.   The t-tuples of each string are compared using the lexicographic ordering on St.

Construction of Hasse Diagrams

  1. Start with the directed graph of the relation in a poset arranging each edge so that an initial edge is below its terminal edge.
  2. Remove the loops that must be present due to reflexivity.
  3. Remove all edges that must be present due to transitivity.
  4. Remove all arrows on the directed edges.

Maximal and Minimal Elements
Maximal
element a in a poset ( S, ≼ ) exists if there is no b ∈ S such that a ≺ b, that is, a is not less than any element of the poset.
Minimal
element a in a poset ( S, ≼ ) exists if there is no b ∈ S such that b ≺ a, that is, a is not greater than any element of the poset.
Greatest Element
of a poset ( S, ≽ ) is one that is greater than every other element.
Least Element
of a poset ( S, ≽ ) is one that is less than every other element.
Least Upper Bound or LUB
element x of the subset A exists if x is an upper bound that is less than every other upper bound of A.
Greatest Lower Bound or GLB
element x of the subset A exists if x is an lower bound that is greater than every other lower bound of A.

Lattices
Lattice
is a poset in which every pair of elements has both a least upper bound and a greatest lower bound.
