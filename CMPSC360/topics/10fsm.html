LANGUAGES AND GRAMMARS


Phrase Structure Grammars
Vocabulary or Alphabet
V is a finite, non-empty set of elements called symbols or letters.
Word or Sentence
over V is a string of finite length of elements of V.
Empty String or Null String
denoted by λ, is the unique string containing no symbols.
V*
is the set of all words over V.
Phrase Structure Grammar or PSG
G = { V, T, S, P ) where
  • V is a finite set of symbols called a vocabulary
  • T ∈ V is a finite set of terminals
  • N = V – T, such that N ∩ T = ∅, is a finite set of non-terminal
  • S ∈ N is the start symbol
  • P = ( ( N ∪ T )* – T* ) x ( N ∪ T )* is a finite set of productions,
    where every production P must contain at least one non-terminal on its left side.

Production
( A, B ) ∈ P where A ∈ ( N ∪ T )* – T* and B ∈ ( N ∪ T )* is denoted by A → B.
Directly Derivable
Let G = ( V, T, S, P ) be a PSG.  
Let w0 = lz0r, which is the concatenation of l, z0, and r, and w1 = lz1r be strings over V.  
If z0 → z1 is a production of G, then w1 is directly derivable from w0 and is denoted by w0 ⇒ w1.  
If w0, w1, . . ., wn are strings over V such that w0 ⇒ w1, w1 ⇒ w2, . . ., wn-1 ⇒ wn,
then wn is derivable from w0 and is denoted by w0 ⇒* wn = w0 ⇒ w1 ⇒ w2 ⇒ . . . ⇒ wn-1 ⇒ wn.
The sequence of steps used to obtain wn from w0 is called a derivation.

Language
generated by PSG G = ( V, T, S, P ), denoted by L ( G ),
is the set of all strings of terminals over V that are derivable form the starting symbol S, i.e.

L ( G ) ∈ V* = { w ∈ T* | S ⇒* w }

Context-Free grammars are used to define the syntax of almost all programming languages.

Notation:

  • Sets of Strings


    • ∅ represents the set with no strings.
    • λ represents a set consisting only of the string with length zero.
      Note that λ is the identity element.
    • Lowercase letters such as a represents a set consisting only of the string with length one.

  • String Operations


    • (L) represents the same set of strings as L.
    • L*, called the Kleene star of L, represents the language
      obtained by concatenating zero or more strings of L, that is,
      L* = λ + L + LL + LLL + . . .
      L* = λ + L + L2 + L3 + . . .
    • LR represents the set of strings a • b or simply ab
      obtained by the concatenation a string a of L with a string b of R.
      Note that concatenation is associative.
    • L + R represents the union of L and R.

  • Hierarchy of Operations

    • ( ) takes precedence over *
    • * takes precedence over •
    • • takes precedence over +

NOTE:   Two grammars G1 and G2 are equivalent if L ( G1 ) = L ( G2 ).

Backus-Naur Form
Backus-Naur Form or BNF
is another notation for specifying grammars where
  • the symbol ::= is used in place of →
  • all non-terminals are delimited with the symbols < >

Extended Backus-Naur Form or EBNF
is an extension of the BNF for specifying grammars where in addition to the specifications of BNF
  • instead of listing all the productions separately, those with the same lefthand non-terminal symbol are combined as one production
  • the righthand side of the combined production separate each production with a bar |

FINITE-STATE MACHINES WITH OUTPUT


Finite-State Machines with Outputs
Finite State Machine or FSM
M is a six–tuple M = ( S, I, O, f, g, s0 ) where
  • S is a finite set of states
  • I is a finite set of input symbols
  • O is a finite set of output symbols
  • f: S x I → S is the next–state function
  • g: S x I → O is the output function
  • s0 ∈ S is the initial state

A FSM may be represented by a state table or by a state diagram.

Recognizes or Accepts
Let M = ( S, I, O, f, g, s0 ) be a FSM and L ∈ I*.
Then M recognizes or accepts L if an input string x belongs to L
iff the last output bit produced by M when given x as input is a 1.

Finite-State Automata
Finite State Automaton or FSA
M is a five-tuple M = ( S, I, f, s0, F ) where
  • S is a finite set of states
  • I is a finite set of input symbols
  • f: S x I → S is the next-state function
  • s0 ∈ S is the initial state
  • F ∈ S is a finite subset of final states or accepting states

Language Recognition by Finite-State Machines
Recognized or Accepted
A string x is said to be recognized or accepted by the fsa M = ( S, I, f, s0, F )
if it takes the initial state s0 to a final state, i.e. f ( s0, x ) is a state in F.
Language Recognized or Language Accepted
The language recognized or language accepted by the fsa M, denoted by L ( M ),
is the set of strings that are recognized by M.  
Two FSA are equivalent if they recognize the same language.

Non-Deterministic Finite-State Automata
Non-Deterministic Finite State Automaton or NDFSA
M = ( S, I, f, s0, F ) where
  • S is a finite set of states
  • I is a finite set of input symbols
  • f: S x I → P ( S ) is the transition function, where P ( S ) is a set of states
  • s0 ∈ S is the initial state
  • F ∈ S is a finite subset of final states

THEOREM01: If the language L is recognized by a NDFSA M0 = ( SM0, IM0, fM0, s0M0, FM0 ),
then L is also recognized by a Deterministic Finite State Automata or DFSA
M1 = ( SM1, IM1, fM1, s0M1, FM1 ).
Proof: The proof describes the construction of the corresponding DFSA.
  • The input set of M1 is the same input set of M0, i.e. IM0 = IM1.
  • Each state in M1 is made up of a set of states in M0.
  • The start state of M1 is s0M1 = { s0M0 }.
  • If f ( siM0, x ) = { si1, si2, . . ., sij, . . ., sik }, then { si1, si2, . . ., sij, . . ., sik } ∈ SM1.
  • If sij ∈ FM0, then { si1, si2, . . ., sij, . . ., sik } ∈ FM1.
Any input string recognized by M0 is also recognized by M1.

LANGUAGE RECOGNITION


Regular Sets
Regular Expressions
over a set V are defined recursively by:
  • The symbol ∅ is a regular expression.
  • The symbol λ is a regular expression.
  • Each a ∈ V is a regular expression.
  • If A and B are regular expressions, then ( A ), AB, A ∪ B, and A* are also regular expressions.

Regular Set
are represented by regular expressions.

Kleene's Theorem
THEOREM02: Kleene's Theorem
A set is regular iff it is recognized by a FSA.

The following defines the construction of a NDFSA for a given string.

  • String a

    • To build the FSA Ma that recognizes a,

    • Designate an initial state sa.
    • Create a state for each necessary transition that would lead to the recognition of the string a.


    Figure 8

  • Concatenation ab

    • Given FSA Ma and Mb.   To build the FSA Mab that will recognize ab,

    • Designate a new initial state sab.
    • Create a transition from sab to the second state of Ma
      with the same transition value from sa to the second state in Ma.
    • Then build a transition from the state prior to the accepting state of Ma to the initial state of Mb
      with the same transition value from the state prior to the accepting state of Ma to the accepting state of Ma.


    Figure 9

  • Union ( a ∪ b ) or ( a + b )

    • Given FSA Ma and Mb.   To build the FSA Ma∪b that will recognize a ∪ b,

    • Designate a new initial state sa∪b.
    • Create a transition from sa∪b to the second state of Ma
      with the same transition value from sa to the second state in Ma.
    • Then create a transition from sa∪b to the second state of Mb
      with the same transition value from sb to the second state in Mb.

  • Kleene Star a*

    • Given FSA Ma.   To build the FSA Ma* that will recognize a*,

    • Designate a new initial state sa*.
    • Create a transition from sa* to the second state of Ma

    • Then create a transition from the accepting state of Ma to the second state of Ma
      with the same transition value from sa to the second state in Ma.

TURING MACHINES


Definition of Turing Machines
Turing Machine
T = ( S, I, f, s0 ) where
  • S is a finite set of states
  • I is a finite set of alphabet containing the blank symbol B
  • f: S x I → S x S x { R, L } is the transition function
  • s0 is the starting state

Transition Rule
of a TM is a five tuple ( s, x, s', x', d ) corresponding to the partial function f ( s, x ) = ( s', x', d )
where s, s' ∈ S and x, x' ∈ I and is interpreted as follows:
  • The Control Unit or CU is in state s and the current cell of the tape contains x.
  • The transition rule indicates that the symbol in the current cell of the tape
    is to be rewritten with x' and the CU goes into state s'.
  • The Read-Write Head or RWH moves right if d = R or moves left if d = L.

Using Turing Machines to Recognize Sets
A TM T = ( S, I, f, s0 ) recognizes a string x ∈ V* ∈ I iff T, starting in the initial position when x is written on the tape, halts in a final state.   T is said to recognize a subset A ∈ V* if x is recognized by T iff x ∈ A.
